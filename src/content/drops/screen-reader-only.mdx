🗣 Screen Reader Only (SRO) é uma técnica utilizada para criação de contextos adicionais via textos que somente leitores de telas irão interpretar.

😀 O legal é que podemos atingir o objetivo proposto apenas com CSS.

<p>
  👉{" "}
  <A href="https://webaim.org/techniques/css/invisiblecontent/#offscreen">
    De acordo com a WebAIM
  </A>
  , basta criarmos a seguinte declaração CSS para posterior reutilização:
</p>

```css
.sr-only {
  position: absolute;
  left: -10000px;
  top: 0;
  width: 1px;
  height: 1px;
  overflow: hidden;
}
```

🤯 E pra exemplificar, imagine botões de Login visualmente fáceis de escolher entre Google, Facebook ou Twitter, porém textualmente não utilizam nenhuma palavra que os dê significado.

![Lista de Botões de Login](https://user-images.githubusercontent.com/1680157/87884573-71133780-c9e5-11ea-9357-a2976375ec4d.png)

😌 Pra diferenciá-los em leitores de telas através da técnica SRO, faríamos algo parecido com o código abaixo:

```html
<ul>
  <li>
    <button>
      <i class="google-logo" aria-hidden="true" />
      <span class="sr-only">Login com Google</span>
    </button>
  </li>
  <li>
    <button>
      <i class="facebook-logo" aria-hidden="true" />
      <span class="sr-only">Login com Facebook</span>
    </button>
  </li>
  <li>
    <button>
      <i class="twitter-logo" aria-hidden="true" />
      <span class="sr-only">Login com Twitter</span>
    </button>
  </li>
</ul>
```

👍 Viu? Prático, simples e bastante útil.
